# Core Beliefs

These beliefs define default decision-making.

1. Keep the cognitive load low for future maintainers.
2. Minimize mutable shared state.
3. Enforce contracts at boundaries, not deep in call stacks.
4. Prefer explicitness over magic.
5. Build observability into the first implementation.
6. Choose reversible designs when uncertainty is high.
7. Pay debt continuously; do not hide it in comments.
